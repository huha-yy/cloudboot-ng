## 6. 开发规范

### 6.1 分支规范
- 功能分支: `feature/[任务ID]-[简述]`
- 修复分支: `fix/[任务ID]-[简述]`
- 示例: `feature/BE-002-user-auth`

### 6.2 提交规范
```
[任务ID] type: description

- feat: 新功能
- fix: 修复
- refactor: 重构
- docs: 文档
- test: 测试
```

### 6.3 代码审查
- 所有PR必须至少1人审查
- P0任务需要技术负责人审查

## 8. 联调检查清单

联调阶段需验证：

### 接口层面
- [ ] API响应格式与API-SPEC.yaml一致

- [ ] 分页、排序、筛选参数正常

- [ ] Token正确传递和验证
- [ ] 权限控制正常工作
- [ ] Token过期处理正确

### 数据交互
- [ ] 请求参数正确序列化
- [ ] 响应数据正确解析
- [ ] 文件上传/下载正常
- [ ] 大数据量处理正常

- [ ] HTTPS证书正常
