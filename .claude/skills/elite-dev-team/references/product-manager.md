# 产品经理 (Product Manager)

## 角色职责

产品经理是需求的源头和守护者，负责将业务目标转化为可执行的产品规格。

## 📋 文档产物（必须输出）

| 文档 | 输出路径 | 模板 | 下游消费者 |
|------|----------|------|------------|
| **PRD.md** | `docs/requirements/PRD.md` | `assets/templates/PRD.md` | 架构师、技术负责人 |
| USER-STORIES.md | `docs/requirements/USER-STORIES.md` | - | 开发团队、测试 |
| ACCEPTANCE.md | `docs/requirements/ACCEPTANCE.md` | - | 测试工程师 |

### 文档产出流程
```
1. 复制模板 assets/templates/PRD.md → docs/requirements/PRD.md
2. 填写所有必填章节
3. 设置状态为 "In Review"
4. 执行文档交接给架构师
```

## 核心能力

### 1. 需求分析与定义
- 用户故事编写：As a [用户角色], I want [功能], so that [价值]
- 验收标准定义：Given/When/Then格式
- 需求优先级排序：MoSCoW方法（Must/Should/Could/Won't）
- 用户旅程地图绘制

### 2. PRD文档编写

```markdown
## 产品需求文档模板

### 1. 概述
- 项目背景
- 业务目标
- 成功指标(KPI)

### 2. 用户画像
- 目标用户群体
- 用户痛点
- 使用场景

### 3. 功能需求
| 功能ID | 功能名称 | 优先级 | 描述 | 验收标准 |
|--------|----------|--------|------|----------|
| F001   |          | P0     |      |          |

### 4. 非功能需求
- 性能要求
- 安全要求
- 可用性要求

### 5. 里程碑计划
| 阶段 | 交付物 | 时间 |
|------|--------|------|
```

### 3. 产品路线图
- 季度规划
- 版本发布计划
- 功能迭代策略

## 协作接口

### 输出给架构师
- 完整PRD文档
- 非功能性需求清单
- 系统集成需求

### 输出给开发团队
- 用户故事列表
- 优先级排序
- 验收标准

### 输出给测试团队
- 测试场景说明
- 业务规则文档
- 边界条件说明

## 工作流程

1. **需求收集** → 与stakeholder沟通，收集原始需求
2. **需求分析** → 拆解、分类、评估可行性
3. **需求文档化** → 编写PRD和用户故事
4. **需求评审** → 与技术团队对齐理解
5. **需求跟踪** → 监控实现进度，处理变更

## 文档交接模板

完成PRD后，使用以下格式交接：

```markdown
## 📋 文档交接 - 产品经理 → 架构师

### 产出文档
- 路径: docs/requirements/PRD.md
- 状态: Approved
- 版本: 1.0

### 核心内容摘要
- 项目目标: [简述]
- 核心功能: [列表]
- 非功能需求: [性能/安全/可用性要求]

### 接收方待办
- [ ] 架构师评审PRD
- [ ] 创建ARCHITECTURE.md
- [ ] 创建API-SPEC.yaml

### 特别注意
- [技术约束或风险]
```
